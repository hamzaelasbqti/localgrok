# Contributing

Contributions are welcome! Here's how you can help improve localgrok.

## Getting Started

1. **Fork** the repository
2. **Create** a feature branch (`git checkout -b feature/amazing-feature`)
3. **Make your changes**
4. **Commit** your changes (`git commit -m 'Add some amazing feature'`)
5. **Push** to the branch (`git push origin feature/amazing-feature`)
6. **Open** a Pull Request

## Development Guidelines

### Code Style

- Follow Kotlin coding conventions
- Use meaningful variable and function names
- Write clear, descriptive commit messages
- Add comments for complex logic
- Keep functions focused and small

### Testing

- Test on multiple Android versions when possible (API 26+)
- Test on different screen sizes
- Verify both themes (Space and Dark) work correctly
- Test with and without network connectivity

### Documentation

- Update README.md if you add new features
- Update relevant docs (SETUP.md, ARCHITECTURE.md, etc.) if needed
- Add code comments for non-obvious logic

### Pull Request Process

1. Ensure your code builds without errors
2. Test your changes thoroughly
3. Update documentation as needed
4. Write a clear PR description explaining what changed and why
5. Reference any related issues

## Areas for Contribution

- Bug fixes
- Performance improvements
- UI/UX enhancements
- Additional theme options
- Accessibility improvements
- Documentation improvements
- Test coverage

## Questions?

Open an issue if you have questions about contributing or need help getting started.
